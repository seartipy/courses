Fundamentals courses

Platform : Any platform is fine, linux recommended.

FUN01. Imperative programming fundamentals using python

Duration : 5 weeks(5 days each week)

Topics : types, expressions, statements, collections, lists, classes, inheritance.

Description : If you wish to learn to program, are familiar with some programming language but didn’t write much code, this is the batch for you. We will slowly learn the basics, give you assignments daily and all the programs you write will be reviewed the next day in the class. This slow paced batch will also teach you OOP, a difficult topic.

FUN02. Functional programming fundamentals using python

Duration : 2 weeks

Topics : functions, closures, lambda expressions, higher order functions, generators

Prerequisites : FUN01

Description : This would be a gentle introduction to functional programming. Only those topics which are relevant to mainstream imperative languages like C#, javascript, C++ and Java are discussed.

FUN03. System fundamentals using python

Duration : 3 weeks

Prerequisites : FUN01

Topics : processes, pipes, files, sockets, asynchronous and non blocking IO.

platform : Linux highly recommended, mac too is fine. Windows cannot be used.

Description : I personally believe these basics are necessary whatever platform or programming language you are using today. Unfortunately only C++ programmers are generally interested in these topics when all programmers especially node developers, developers dealing with concurrency or even web developers will immensely benefit. This introductory course will teach you operating systems and network programming in the simplest manner possible without sacrificing too many details. python is an excellent language to avoid discussing unnecessary details, still managing to access system calls directly.

FUN04. Data structures using python

Duration : 3 weeks

Prerequisites : FUN01

Topics : binary search, insertion sort, merge sort, quick sort, single and double linked lists, avl trees, hash tables

Description : This will be an introductory course discussing only those topics which are required for any working professional. Seperate courses will be offered for those who are interested in the topic.

FUN05. concurrency using jython or ironpython

Duration : 2 Weeks

Prerequisites : FUN02

Topics : threads, synchronization, mutual exclusion

Description : Extremely complex topic. Basics are necessary even if you don’t generally deal with concurrency directly. If you are completely new to multithreading or need just the basics to understand your platform better, start here.

FUN06. Web server side fundamentals using flask(python) or sinatra(ruby)

Duration : 3 Weeks

Prerequisites : FUN01 Topics : HTTP, REST, session handling, views, routing, testing

Description : Whatever be the server side framework you use, most ideas, concepts are similar. This will teach you all the important topics you need for server side web development framework using sinatra or flask. If you are a C++ programmer in need of a simple web framework, or a beginner developer planning to learn some framework, this course will get you started quickly.

FUN07. Web client side fundamentals

Duration : 4 weeks

Prerequisites : FUN01

Topics : javascript(functions, objects, prototype, inheritance, modules), jquery(DOM and ajax), unit testing

Description : All you need to get started on client side web development. You need to be an experienced developer, this is not for absolute beginners. Javascript is surprisingly a very good language at heart, this course will discuss why. You will be a pro at javascript after mastering this course.

FUN08. Single Page Applications using backbonejs

Duration : 2 weeks

Prerequisites : FUN01, FUN07

Topics : models, views, collections, routes, REST, templating

Description : We will learn most important concepts for creating SPAs using backbonejs, a simple MV* framework. SPAs are extremely important with many web applications as feature rich as desktop applications. Nobody should use backbonejs today for real applications, as better frameworks exist. If you need to, use marionette instead, a framework built on backbonejs.
